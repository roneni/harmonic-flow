1 Getting Started

Install Claude Code (native installer — no Node.js needed)

# Recommended (auto-updates)
curl -fsSL https://claude.ai/install.sh | bash

# macOS Homebrew
brew install --cask claude-code

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex

# Windows WinGet
winget install Anthropic.ClaudeCode

Requirements: macOS 13+, Windows 10+, Ubuntu 20.04+, 4GB+ RAM

cd your-project
claude
/init

3 Memory File Hierarchy
~/.claude/CLAUDE.mdGlobal — all projects
/CLAUDE.mdProject root — shared on git
./frontend/CLAUDE.mdSubfolder — scoped content (lazy-loaded)
CLAUDE.local.mdPersonal overrides — gitignored
~/.claude/projects/<p>/memory/Auto-memory — Claude writes for itself
Managed Policy (org-wide, read-only)

macOS: /Library/Application Support/ClaudeCode/CLAUDE.md
Linux: /etc/claude-code/CLAUDE.md

Conflict Resolution
  • Keep each file <200 lines
  • More specific levels WIN on conflicts
  • Subfolder overrides parent
  • Ancestors load at startup; descendants lazy-load
  • Never overwrite parent context
4 CLAUDE.md Best Practices
  • Run /init first then refine output
  • Be specific in instructions
  • Add gotchas Claude cannot infer from code
  • Reference docs with @filename
  • Add workflow rules
  • Keep memory concise — every line costs context
  • Commit to Git for team sharing
  • Use .claude/rules/ for file-scoped rules (glob patterns)
File-Scoped Rules
# .claude/rules/api-routes.md
---
paths:
  - "src/api/**/*.ts"
---
All endpoints must include input validation
All responses use standard error format

# Testing Patterns
Use describe + it + AAA pattern

7 MCP (Model Context Protocol)

Connect Claude Code to external tools, databases, and APIs.

# Add HTTP server (remote)
claude mcp add --transport http \
  serverName https://api.example.com/mcp

# Add stdio server (local)
claude mcp add --transport stdio \
  serverName -- npx package-name

# Add with auth
claude mcp add --transport http \
  --header "Authorization: Bearer TOKEN" \
  serverName URL

# List / manage
claude mcp list
/mcp  # in-session

9 Permissions & Safety
Permission Modes
defaultPrompts on first use
acceptEditsAuto-accepts file edits
planRead-only mode
dontAskAuto-denies unless pre-approved
bypassPermissionsSkips ALL checks (danger)
{
  "permissions": {
    "allow": ["Bash(npm run lint:*)"],
    "ask":   ["Bash(git push *)"],
    "deny":  ["Read(.env)",
              "Bash(rm -rf *)"]
  }
}

Eval order: deny → ask → allow

Settings Precedence

Managed → CLI flags → Local project → Shared project → User

15 Multi-File Editing & Testing
Multi-File Workflow
  • Plan first — Shift+Tab into Plan Mode
  • Read before edit — always read a file first
  • Subagents for parallel edits — isolation: worktree
  • Prefer Edit over Write — Edit sends diffs only
  • Use @filename — attach files for context
  • /diff to review — check all changes
  • Commit atomically — one logical change per commit
Testing Strategies
  • Create .claude/skills/testing/SKILL.md
  • AAA pattern — Arrange, Act, Assert
  • Use PreToolUse hooks for CI validation
  • Factory mocks over inline mocks
  • Always test error paths, not just happy paths
  • Pre-approve: "allow": ["Bash(npm run test*)"]
